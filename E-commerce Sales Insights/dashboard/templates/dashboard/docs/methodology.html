Methodology

Data Pipeline

  1. Ingestion: CSV files are uploaded via the secure Datasets dashboard.
  2. Validation & Cleaning (Cleaning Engine): The system uses Pandas to standardize column names. It validates the presence of required columns, checks for invalid date formats, and flags negative revenue values to prevent skewed analytics.
  3. Storage: The clean file is stored on disk, and its metadata is saved to the SQLite/PostgreSQL database.
  4. Processing (Analytics Engine): The active dataset is loaded into memory. When the user selects filters, Pandas applies these boolean masks dynamically.
  5. KPI Generation: Key metrics like Gross Profit Margin, Return Rates, and AOV are aggregated from the filtered dataframe.
  6. Visualization (Chart Engine): The filtered dataframe is passed to Plotly Express, generating JSON schema configurations for rich client-side rendering.

Insight Engine (AI-Free Logic)

To guarantee deterministic and reliable business advice without the overhead or hallucination risks of LLMs, the platform utilizes a rule-based insight generator.

It operates using boundary thresholds. For example, if cancellation_rate > 15%, it triggers a Critical Risk Alert string. If returning_customer_rate >= 30%, it logs a Good Customer Insight. This ensures executives always get fast, verifiable summaries.

Machine Learning Forecasting

The Forecasting module relies on Simple Linear Regression via the scikit-learn library.

  • Historical transaction dates are grouped by month (dt.to_period('M')).
  • Revenue and quantities are summed for each period.
  • A time-step index (0, 1, 2...) is created and used as the independent variable (X).
  • The target metric (Revenue, Product Demand) acts as the dependent variable (y).
  • The model is fitted and used to predict the next 3 time-steps, with values clamped to a minimum of 0.
