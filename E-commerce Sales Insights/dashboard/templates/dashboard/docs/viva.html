A: The main purpose is to provide an enterprise-ready SaaS platform that ingests e-commerce CSV data, cleans it, calculates KPIs, and generates interactive visualizations and ML-based forecasts, helping business users make informed decisions.

A: I used Django because it is a high-level Python framework that encourages rapid development and clean design. It comes with built-in security features, an ORM, and integrates seamlessly with Python data science libraries like Pandas.

A: The charts are generated using Plotly. Plotly creates JSON configurations in Python, which are then parsed and rendered dynamically on the frontend using Plotly.js, allowing for tooltips, zooming, and downloading without page reloads.

A: It uses HTMX. When a user changes a filter in the sidebar, a background request is sent to the server. The server processes the filtered dataframe and returns a partial HTML response containing the updated charts and tables, which HTMX swaps into the DOM seamlessly.

A: Simple Linear Regression from the scikit-learn library. It maps historical monthly revenues against a time index to plot a trend line and project future values.

A: Instead of relying on expensive or unpredictable Generative AI, the Insight Generator uses a rule-based engine. It analyzes the calculated KPIs (like cancellation rates > 15% or negative monthly growth) and constructs human-readable sentences using standard Python if/else logic.

A: The cleaning_engine.py script automatically standardizes column headers, fills or drops missing values based on data types, drops duplicates, and validates that all required columns are present. It also flags incorrect data types and invalid dates.
